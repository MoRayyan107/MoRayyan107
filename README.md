### рҹ’¬ Discord вҖ” Real-time Messaging Service 
> `Java` `Concurrent I/O` `Event-driven Architecture` `Socket Programming`

**The Challenge:**
Build a distributed, multi-user messaging service capable of handling simultaneous connections across multiple server instances with real-time message propagation.

**The Solution:**
Implemented a server architecture using Java's concurrency primitives вҖ” replacing raw thread-per-client with a fixed ExecutorService thread pool, and decoupling message delivery from sending via BlockingQueue with dedicated dispatcher threads. Extended to a distributed system using Kafka for cross-server messaging and Redis for shared group state and user routing.

```
Client (TCP Socket)
        вҶ“
ClientHandler (Thread Pool вҖ” 200 threads)
        вҶ“
SafeGroupChat (BlockingQueue + dispatcher thread)
        вҶ“
KafkaProducer вҶ’ KafkaConsumer (For Cross-server Messaging)
        вҶ“
Redis (group membership, user-server routing)
        вҶ“
Target ClientHandler вҶ’ sendToClient()
```
